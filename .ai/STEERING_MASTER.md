# プロジェクトルール

プロジェクト接頭語については [PREFIX_RULES.md](./PREFIX_RULES.md) を参照すること。

## 命名規則

### データベース

- テーブル名: `{プロジェクト接頭語}_<テーブル名>`

### AWS、GCP

- リソース作成時: タグにプロジェクト接頭語を付与すること
- リソース操作時: タグにプロジェクト接頭語が付いているリソースのみを対象とすること

## コマンド

### 「おはよう」コマンド

ユーザーが「おはよう」と入力した場合、以下を順に実行する:

1. プロジェクトの概要を説明する（構成、使用技術、ナレッジベースの内容など）
2. Git の使用有無を確認し、使用中ならアカウント名とリポジトリ名を表示する
3. AWS の使用有無を確認し、使用中ならアカウント情報を表示する
4. Firebase の使用有無を確認し、使用中ならアカウント情報を表示する
5. Google Cloud の使用有無を確認し、使用中ならアカウント情報を表示する
6. デプロイ先のアカウント情報と URL を取得・表示する


# AI Behavior Rules

- ユーザーから「質問」のみが投げかけられた場合は、状況の説明や解決策・選択肢の提示のみを行い、ユーザーから「実装してください」と明確な指示があるまで勝手なコード実装・ファイル編集を行わないこと。
